File Management Architecture

  • Project mode: Scans configured project directory recursively for .hdbcalculationview files. Shows file path, parsed metadata (CUBE/DIMENSION, node count), last modified time.
  • HANA Runtime mode: Queries SYS.OBJECTS + _SYS_BI.BIMC_ALL_CUBES to list deployed Calculation Views. Opens by reading the XML definition from _SYS_REPO.ACTIVE_OBJECT (classic) or HDI container metadata.
  • Data source picker (Database tab): Uses hana-cli's existing tables / views commands via REST API. Filters by schema, type, name pattern. Shows column count.
  • Data source picker (Project tab): Lists local .hdbtable, .hdbview, .hdbcalculationview files. Parsed to show column info.
  • Save behavior: Project files save in-place (or "Save As" to new path). Runtime views always "Export As" to local file.
  • Dirty state: Unsaved changes shown with dot indicator on tab. Ctrl+S saves. Close with unsaved prompts confirmation.
