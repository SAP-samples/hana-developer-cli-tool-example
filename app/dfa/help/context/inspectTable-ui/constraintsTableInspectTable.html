Table Constraint Listing

This table displays all constraints defined on the inspected table. Constraints are rules that enforce data integrity and define relationships between tables.

The Constraints tab shows various types of constraints including:

  • Primary Key: Uniquely identifies each row in the table
  • Foreign Key: Enforces referential integrity with other tables
  • Unique: Ensures column values are unique across rows
  • Check: Validates data based on specified conditions
  • Not Null: Prevents NULL values in specific columns

For each constraint, you can see the constraint name, type, involved columns, and any referenced tables (for foreign keys). This information is essential for understanding data validation rules and table relationships.
